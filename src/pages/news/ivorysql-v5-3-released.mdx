    <div className="post-content">
      <h2 id="version-overview">Version Overview</h2>
      <p>
        On March 12, 2026, the IvorySQL community officially released IvorySQL 5.3.
      </p>
      <p>
        Built on PostgreSQL 18.3, this release keeps pace with the latest stable PostgreSQL capabilities while further strengthening cloud-native deployment, ecosystem extensibility, and overall user experience. IvorySQL 5.3 also introduces an online trial environment so users can explore it directly in the browser without local installation.
      </p>
      <p>
        For the complete update list, visit the official documentation site at <a href="https://docs.ivorysql.org/" target="_blank" rel="noopener noreferrer">docs.ivorysql.org</a>.
      </p>

      <h2 id="pg-18-3">Enhanced Features from PostgreSQL 18.3</h2>
      <ul>
        <li>Fixed failures that could occur after replaying multixid truncation WAL records generated by older minor versions.</li>
        <li>Avoided incorrect invalid-encoding errors when applying <code>substring()</code> to toasted data.</li>
        <li>Addressed an omission in the fix for CVE-2026-2007.</li>
        <li>Corrected the volatility markings of <code>json_strip_nulls()</code> and <code>jsonb_strip_nulls()</code>.</li>
        <li>Fixed the computation of outer joins that can produce null values in <code>LATERAL UNION ALL</code> subquery outputs.</li>
      </ul>
      <p>
        For more details, see the official <a href="https://www.postgresql.org/docs/release/18.3/" target="_blank" rel="noopener noreferrer">PostgreSQL 18.3 Release Notes</a>.
      </p>

      <h2 id="pg-18-2">Enhanced Features from PostgreSQL 18.2</h2>
      <ul>
        <li>Guarded against unexpected dimensions in <code>oidvector</code> and <code>int2vector</code> values.</li>
        <li>Improved selectivity estimators to avoid attaching them to operators that accept unexpected data types.</li>
        <li>Fixed a buffer overflow in PGP decryption functions in the <code>contrib/pgcrypto</code> module.</li>
        <li>Fixed insufficient validation of multibyte character lengths.</li>
        <li>Enhanced the <code>contrib/pg_trgm</code> module to cope with changes in string lowercasing behavior.</li>
      </ul>
      <p>
        For more details, see the official <a href="https://www.postgresql.org/docs/release/18.2/" target="_blank" rel="noopener noreferrer">PostgreSQL 18.2 Release Notes</a>.
      </p>

      <h2 id="ivory-features">New Features of IvorySQL 5.3</h2>
      <ul>
        <li>Upgraded to the PG 18.3 kernel.</li>
        <li><a href="http://trial.ivorysql.org" target="_blank" rel="noopener noreferrer">Online Trial</a>: a <a href="https://github.com/IvorySQL/ivorysql-wasm/pull/7" target="_blank" rel="noopener noreferrer">browser-based interactive environment</a> that lets users explore and test IvorySQL 5.3 without local installation.</li>
        <li>Full-platform installation package support across X86, ARM, MIPS, and LoongArch architectures.</li>
        <li>Containerized deployment support with one-click delivery of standalone or high-availability IvorySQL 5.3 clusters through Docker Compose, Podman, Swarm, Helm, and Operator.</li>
        <li>IvorySQL Cloud 5.3 released with a fully managed visual control plane for subscription management, lifecycle operations orchestration, and ecosystem service integration.</li>
        <li>Support for 10 additional PostgreSQL extensions: <code>pg_cron</code>, <code>pgAudit</code>, <code>PostGIS</code>, <code>pgRouting</code>, <code>PGroonga</code>, <code>ddlx</code>, <code>pgsql-http</code>, <code>system_stats</code>, <code>plpgsql_check</code>, and <code>pgvector</code>.</li>
      </ul>

      <h2 id="fixed-issues">Fixed Issues</h2>
      <ul>
        <li>Cleaned up compilation warnings during the build process.</li>
        <li>Fixed the <code>raw_expression_tree_walker</code> handling issue for <code>ColumnRefOrFuncCall</code>.</li>
        <li>Fixed failures when upgrading from PostgreSQL to IvorySQL.</li>
        <li>Improved PL/iSQL error messages when <code>ivorysql.compatible_mode = 'pg'</code>.</li>
        <li>Fixed an out-of-bounds read in <code>token_is_col_id()</code> within <code>liboracle_parser.c</code>.</li>
        <li>Fixed the issue where psql tab completion incorrectly displayed <code>INVISIBLEINCREMENT</code>.</li>
        <li>Fixed variable reset issues in nested function expressions.</li>
        <li>Fixed type coercion failures when packages and subprograms mixed positional and named arguments.</li>
        <li>Fixed a potential segmentation fault when package procedures initialized local variables from parameters.</li>
        <li>Fixed regression test failures caused by cross-year execution.</li>
      </ul>
