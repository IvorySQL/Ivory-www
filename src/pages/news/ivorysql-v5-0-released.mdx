          Built on a full upgrade to PostgreSQL 18.0, IvorySQL 5.0 delivers groundbreaking enhancements to Oracle compatibility, upgraded ecosystem component integrations, and in-depth cloud-native support.
        </p>

        <h2 id="pg-18">Powered by PostgreSQL 18.0</h2>
        <ul>
          <li>Introduced an asynchronous I/O (AIO) subsystem for significant performance improvements.</li>
          <li>The <code>pg_upgrade</code> tool: Added a new feature to preserve optimizer statistics.</li>
          <li>Supported skip scan for multi-column B-tree indexes.</li>
          <li>New <code>uuidv7()</code> function for time-ordered UUIDs.</li>
          <li>Virtual generated columns now supported as the default type.</li>
          <li>Supported OAuth authentication.</li>
          <li>Enhanced RETURNING clause with OLD and NEW keywords.</li>
          <li>Time range constraints for PRIMARY KEY, UNIQUE, and FOREIGN KEY.</li>
        </ul>
        <p>For more details, see the <a href="https://www.postgresql.org/docs/release/18.0/" target="_blank" rel="noopener noreferrer">PostgreSQL 18.0 Release Notes</a>.</p>

        <h2 id="oracle-compat">21 New Oracle Compatibility Features</h2>
        <p>IvorySQL 5.0 brings a massive leap in migration capabilities:</p>
        <ul>
          <li>Oracle-compatible ROWID support for seamless tool migration.</li>
          <li>PL/iSQL %ROWTYPE and %TYPE support to simplify coding.</li>
          <li>Empty string to NULL conversion to comply with Oracle compatibility rules.</li>
          <li>Oracle-compatible <code>SYS_GUID</code>, <code>SYS_CONTEXT</code>, and <code>USERENV</code> functions.</li>
          <li>Extended OUT parameter support for libpq, stored procedures, and functions.</li>
          <li>Nested subprograms with support for overloading.</li>
          <li>Oracle-compatible <code>INSTR</code> function and <code>FORCE VIEW</code> option.</li>
        </ul>

        <h2 id="cloud-native">Cloud-Native & Containerization</h2>
        <ul>
          <li><strong>Kubernetes (K8s):</strong> Deploy IvorySQL standalone or high-availability clusters via Helm.</li>
          <li><strong>IvorySQL Operator v5:</strong> Adapted for version 5.0 with synchronized component upgrades.</li>
          <li><strong>IvorySQL Cloud 5.0:</strong> A visual managed control plane for the full lifecycle management.</li>
          <li><strong>Multi-Platform:</strong> Packages covering X86, ARM, MIPS, and LoongArch architectures.</li>
        </ul>

        <h2 id="fixed-issues">Fixed Issues</h2>
        <ul>
          <li>Resolved accurate OID conflict detection in directory tools.</li>
          <li>Eliminated dangling pointer scenarios in the MainLoop for high concurrency.</li>
          <li>Fixed test framework assumptions to re-enable end-to-end Oracle compatibility tests.</li>
          <li>Corrected global unique index enforcement logic across all platforms.</li>
        </ul>
