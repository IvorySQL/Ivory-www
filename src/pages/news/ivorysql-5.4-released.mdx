    <div className="post-content">
      <p>
        On June 17, 2026, the IvorySQL community officially released <strong>IvorySQL 5.4</strong>. As an Oracle-compatible open-source PostgreSQL database, IvorySQL 5.4 is built on <strong>PostgreSQL 18.4</strong> and includes kernel updates, feature enhancements, and ecosystem expansion, further advancing AI, cloud-native deployment, and database management capabilities.
      </p>
      <p>Full release notes: <a href="https://docs.ivorysql.org/" target="_blank" rel="noopener noreferrer">docs.ivorysql.org</a></p>

      <h2 id="kernel">PostgreSQL 18.4 Kernel Update</h2>
      <ol>
        <li>Prevent unbounded recursion while processing startup packets</li>
        <li>Fix assorted integer overflows in memory-allocation calculations</li>
        <li>Properly quote subscription names in <code>pg_createsubscriber</code></li>
        <li>Properly quote object names in logical replication origin checks</li>
        <li>Reject over-length options in <code>ts_headline()</code></li>
        <li>Detect faulty input when restoring attribute MCV statistics</li>
      </ol>

      <h2 id="features">Key Enhancements</h2>

      <h3>Upgrade to PG 18.4 kernel</h3>
      <p>PostgreSQL kernel upgraded to version 18.4 for improved stability, security and compatibility.</p>

      <h3>IvorySQL Skills</h3>
      <p><a href="https://github.com/IvorySQL/skills" target="_blank" rel="noopener noreferrer">github.com/IvorySQL/skills</a></p>
      <p>Empowering AI assistants to become IvorySQL database experts.</p>

      <h3>Online Experience</h3>
      <p><a href="http://trial.ivorysql.org/" target="_blank" rel="noopener noreferrer">trial.ivorysql.org</a></p>
      <p>An interactive, browser-based environment is now available, allowing users to explore and evaluate IvorySQL 5.4 in real time — no installation required.</p>

      <h3>Packaging for All Platforms</h3>
      <p>Provides multi-platform media packages for <strong>X86, ARM, MIPS, and LoongArch</strong> architectures.</p>

      <h3>Containerized Deployment Support</h3>
      <p>Supports one-click deployment of standalone or highly available IvorySQL 5.4 clusters via <strong>Docker Compose, Podman, Swarm, Helm</strong>, and the <strong>IvorySQL 5.4 Operator</strong>.</p>

      <h3>IvorySQL Cloud 5.4</h3>
      <p>A fully managed, graphical control plane that handles IvorySQL 5.4 database subscriptions, orchestrates end-to-end lifecycle operations, and integrates surrounding ecosystem services.</p>

      <h3>PostgreSQL Extensions</h3>
      <p>Added support for additional PostgreSQL extensions and tools:</p>
      <ul>
        <li><strong>AI & Vector</strong>: pg_ai_query, pgvector</li>
        <li><strong>Graph</strong>: AGE</li>
        <li><strong>GIS & Routing</strong>: PostGIS, pgRouting</li>
        <li><strong>Chinese text search</strong>: pg_jieba, Zhparser, pg_bigm</li>
        <li><strong>Full-text search</strong>: PGroonga, pg_textsearch</li>
        <li><strong>Scheduling</strong>: pg_cron, pgagent</li>
        <li><strong>Monitoring</strong>: system_stats, pg_stat_monitor, pg_show_plans</li>
        <li><strong>SQL tools</strong>: ddlx, plpgsql_check, pg_hint_plan</li>
        <li><strong>HTTP</strong>: pgsql-http, pg_curl</li>
        <li><strong>Connection pool</strong>: PgBouncer</li>
        <li><strong>Partitioning</strong>: pg_partman</li>
        <li><strong>WAL</strong>: wal2json</li>
        <li><strong>Foreign data</strong>: redis_fdw</li>
      </ul>

      <h3>SQL Compatibility</h3>
      <p>Enhanced the EXTRACT function support to accept additional syntax variations in SQL queries.</p>
