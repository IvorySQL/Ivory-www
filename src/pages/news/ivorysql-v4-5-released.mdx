    <div className="post-content">
      <p>IvorySQL 4.5 was officially released on June 4, 2025!</p>
      <p>This new version offers full support for PostgreSQL 17.5, introduces a host of new features, and addresses known issues.</p>

      <h2 id="enhanced-features">Enhanced Features</h2>
      <p>Enhancements from PostgreSQL 17.5:</p>
      <ul>
        <li>Fixed the single-byte buffer overread issue when validating invalid strings declared with GB18030 encoding</li>
        <li>Properly handled self-referential foreign keys on partitioned tables</li>
        <li>Prevented data loss when merging compressed BRIN summaries via <code>brin_bloom_union()</code></li>
        <li>Corrected the parsing of external CTE names referenced by <code>INSERT/UPDATE/DELETE/MERGE</code> commands nested in WITH clauses</li>
        <li>Rectified the processing logic of <code>ALTER TABLE ADD COLUMN</code> for domain types with default values</li>
      </ul>
      <p>For more details, refer to the <a href="https://www.postgresql.org/docs/release/17.5/" target="_blank">PostgreSQL Release Notes</a>.</p>

      <h2 id="new-features">New Features</h2>
      <ul>
        <li>Upgraded to the PostgreSQL 17.5 kernel</li>
        <li><strong>Full-platform MIPS packaging support:</strong> In addition to the existing ARM and X86 architectures, multi-platform media packages for the MIPS architecture are now available, compatible with mainstream domestic and international operating systems (including Kylin, UOS, NSAR OS, etc.)</li>
        <li><strong>IvorySQL Online Experience:</strong> A web-based instant experience environment is provided, enabling users to operate the IvorySQL 4.5 database directly via a browser</li>
        <li>New Code of Conduct added</li>
        <li>Community Contribution Guidelines updated</li>
        <li>Implemented automatic document building and official website updates via PR</li>
        <li>Enhanced contributor workflow: Supported self-service task assignment with the <code>/assign</code> command</li>
        <li>IvorySQL Operator V4 adapted for IvorySQL 4.5, with system components and database extension versions upgraded</li>
      </ul>
      <p>For more information about the new IvorySQL Online Experience feature, read the article <a href="https://mp.weixin.qq.com/s/BdZ0nWuFWFx8gnpbBVV42g" target="_blank">IvorySQL-WASM: A No-Install Journey of Database Exploration</a>.</p>

      <h2 id="resolved-issues">Resolved Known Issues</h2>
      <ul>
        <li>Fixed the enumeration value error of <code>check_compatible_mode</code> and <code>assign_compatible_mode</code></li>
        <li>Corrected the description of GUC framework test results in the documentation</li>
        <li>Rectified the description of Oracle compatibility parameters in the documentation</li>
        <li>Fixed the README path issue in the documentation</li>
        <li>Resolved the problem where <code>search_path</code> failed to update when the compatibility mode was changed</li>
        <li>Fixed compatibility issues between <code>ModifyTableContext</code> and <code>UpdateContext</code></li>
        <li>Addressed the ineffectiveness of the <code>initdb -C identifier_case_switch</code> command</li>
      </ul>
      <p>For full details, see the <a href="https://github.com/IvorySQL/IvorySQL/releases/tag/IvorySQL_4.5" target="_blank">IvorySQL 4.5 Release Notes</a>.</p>

      <h2 id="source-code">Source Code</h2>
      <p>IvorySQL mainly consists of two code repositories:</p>
      <ul>
        <li>IvorySQL Database Source Code: <a href="https://github.com/IvorySQL/IvorySQL" target="_blank">https://github.com/IvorySQL/IvorySQL</a></li>
        <li>IvorySQL Official Website: <a href="https://github.com/IvorySQL/Ivory-www" target="_blank">https://github.com/IvorySQL/Ivory-www</a></li>
      </ul>
